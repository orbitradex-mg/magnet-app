# Исправление: "Not Found" при обновлении страницы

## Проблема

При обновлении страницы (F5 или Cmd+R) появляется ошибка "Not Found", хотя при переходе по ссылкам все работает.

## Причина

React Router работает на клиенте, но при обновлении страницы браузер пытается найти файл на сервере по пути (например, `/admin`), которого не существует.

## Решение

Я создал файл `_redirects` в папке `client/public/`. Этот файл говорит Render перенаправлять все запросы на `index.html`.

---

## Что нужно сделать:

### 1. Файл уже создан

Файл `client/public/_redirects` уже создан с содержимым:
```
/*    /index.html   200
```

### 2. Задеплоить изменения

Выполните в терминале:

```bash
cd /Users/romanbilskiy/Documents/Magnet_app
git add .
git commit -m "Fix: добавлен редирект для React Router"
git push
```

### 3. Дождаться деплоя

- Render автоматически задеплоит изменения (2-3 минуты)
- После деплоя обновление страницы будет работать!

---

## Проверка

После деплоя:
1. Откройте ваш сайт
2. Перейдите на любую страницу (например, `/admin`)
3. Нажмите F5 (или Cmd+R на Mac)
4. Страница должна обновиться без ошибки!

---

## Как это работает

Файл `_redirects` говорит Render:
- Все запросы (`/*`) перенаправлять на `/index.html`
- С кодом ответа 200 (успешно)
- React Router на клиенте обработает маршрут
